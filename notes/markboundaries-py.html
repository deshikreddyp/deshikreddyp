mark_boundaries.py

Category: FEniCS

Last updated: ...

In complex CFD simulations, especially those involving patient-specific geometries, manually tagging boundaries (inlet, outlet, walls) is impractical. The mark_boundaries.py utility automates this by grouping facets based on the angle between their normals.

Algorithm Overview.

The script identifies "feature edges" where the angle between neighboring facets exceeds a specified threshold (e.g., $30^\circ$). It then uses a region-growing approach to assign unique physical tags to continuous surface patches.

Implementation Details

1. Vector Math utilities

import numpy as np

def angle_between(v1, v2):
    """Compute the angle in radians between two vectors."""
    v1 = v1 / np.linalg.norm(v1)
    v2 = v2 / np.linalg.norm(v2)
    angle = np.arccos(abs(np.clip(np.dot(v1, v2), -1.0, 1.0)))
    return angle

2. Region Growing Loop

def _mark_loop(bmesh, normals, threshold, boundaries, queue, visited, marker):
    # Iterate through facets in the queue and grow the region
    while not queue.is_empty():
        facet = queue.pop()
        visited[facet] = True
        boundaries[facet] = marker
        
        N = normals[facet.index()]
        for neighbor in facet.neighbors():
            if not visited[neighbor]:
                n = normals[neighbor.index()]
                if angle_between(N, n) < threshold:
                    queue.append(neighbor)
Normal Consistency.

This method assumes all facets in the STL/BRep are consistently oriented. If normals are flipped, the abs() in dot(v1, v2) prevents unintended feature edges, but it's still best to fix orientation first.

Cleanup Logic.

The full script includes post-processing to merge small "orphan" patches (e.g., $< 10$ facets) into their largest neighbor to ensure clean boundary definitions.

LaTeX Plug-in for Inkscape

Category: Tools · LaTeX · Scientific Illustration

Last updated: ...

For scientific publications, it is often necessary to render LaTeX equations directly inside Inkscape vector graphics. This ensures that labels, axes, and annotations match the font and style of the main document exactly.

Recommended Extension: TexText.

While Inkscape has a built-in "LaTeX (pdflatex)" input, TexText is the industry standard. It allows for non-destructive editing (re-opening existing LaTeX objects) and supports complex preambles with packages like amsmath, physics, and bm.

Installation (Linux/Cluster)

If you are working on a managed workstation, ensure you have a working TeX distribution and the python-lxml dependency:

# Essential dependencies
sudo apt-get install inkscape texlive-full pstoedit python3-lxml

Usage Workflow

  1. Open Inkscape and go to Extensions > Text > TexText.
  2. Enter your LaTeX code (e.g., $ \nabla \times \mathbf{E} = -\frac{\partial \mathbf{B}}{\partial t} $).
  3. Configure the preamble if you need specific symbols or fonts.
  4. Click Apply. The equation is added as a grouped vector path.
Rendering Failures.

If equations fail to appear, check the inkscape.log. The most common cause is missing pstoedit or ghostscript in the system path, which are required for converting PDF/PS output back into SVG paths.

Pro Tip.

To edit an existing TexText object, select it and launch the extension again. Your original LaTeX code will be pre-populated in the dialog box.

Fluid Structure Interaction — ALE

Technical Notes on FSI category · Mesh motion · ALE framework

Last updated: ...

Overview

Fluid-structure interaction (FSI) modelling broadly falls into two categories based on how the governing equations are solved:

  1. Monolithic approach — fluid, solid, and mesh motion equations are assembled into a single system and solved simultaneously.
  2. Partitioned (non-monolithic) approach — the fluid and solid solvers are run separately and coupled at the interface iteratively.

There are also several approaches based on how the fluid-structure interface is represented:

  • Body-fitted method — mesh conforms to the structure boundary; typically used with the ALE framework.
  • Immersed boundary method — the structure is embedded in a fixed background fluid mesh; interface conditions imposed via body forces.
  • Mesh-free method — no background mesh; particles or radial basis functions carry solution data.

The interface coupling itself can further be classified by the type of mesh matching:

  1. Strong (conformal meshes) — fluid and solid meshes share nodes at the interface; velocity and traction continuity are enforced exactly.
  2. Weak / Nitsche method (non-conformal meshes) — interface conditions are enforced weakly via a penalty-like variational term.
Quasi-Direct Coupling.

My own work uses the monolithic + body-fitted + fully coupled strategy with quasi-direct coupling for stability and performance.

Why ALE? Lagrangian vs. Eulerian

In classical fluid mechanics we work in an Eulerian frame: the computational domain is fixed in space and fluid flows through it. This is natural for fluids, which undergo large deformations and flow, but it makes tracking solid deformation awkward.

Solids, on the other hand, are naturally described in a Lagrangian frame: the mesh moves with the material. Using an Eulerian frame for solids would obscure the material identity needed for constitutive modelling.

The Arbitrary Lagrangian-Eulerian (ALE) framework accommodates both: the computational mesh moves with the solid boundary (Lagrangian) but deforms smoothly in the fluid interior. The governing equations gain an additional mesh velocity term that accounts for the moving reference frame.

ALE Solution Procedure

At each time step, an ALE-FSI solver performs two coupled sub-problems:

  1. Mesh motion problem. Given interface displacement, extend mesh deformation into the fluid domain: \[ \nabla \cdot (\mu_m \nabla \mathbf{w}) = \mathbf{0} \quad \text{in} \quad \Omega_f \] where $\mathbf{w}$ is the mesh displacement field and $\mu_m$ is a stiffening parameter.
  2. Moving-domain problem. The fluid momentum equation acquires an ALE convection term involving the mesh velocity $\dot{\mathbf{w}}$: \[ \rho \left( \frac{\partial \mathbf{u}}{\partial t} \bigg|_\chi + (\mathbf{u} - \dot{\mathbf{w}}) \cdot \nabla \mathbf{u} \right) = -\nabla p + \mu \nabla^2 \mathbf{u} \]
Key Distinction.

In the solid domain, mesh displacement equals material displacement. In the fluid domain, mesh motion has no physical meaning; it is a purely computational device to handle the moving boundary.

Coupling Strategies

  1. Block-iterative (Gauss-Seidel) — Fluid and solid are solved in sequence. Robust but prone to added-mass instabilities.
  2. Quasi-direct coupling — Approximates the coupled Jacobian. Faster convergence than iterative methods.
  3. Direct (monolithic) — Unified system matrix solved simultaneously. Optimal convergence but complex implementation.

Fluid Structure Interaction — ALE

Fluid mechanics · FSI · Mesh motion · ALE framework

Overview

Fluid-structure interaction (FSI) modelling broadly falls into two categories based on how the governing equations are solved:

  1. Monolithic approach — fluid, solid, and mesh motion equations are assembled into a single system and solved simultaneously.
  2. Partitioned (non-monolithic) approach — the fluid and solid solvers are run separately and coupled at the interface iteratively.

There are also several approaches based on how the fluid-structure interface is represented:

  • Body-fitted method — mesh conforms to the structure boundary; typically used with the ALE framework.
  • Immersed boundary method — the structure is embedded in a fixed background fluid mesh; interface conditions imposed via body forces.
  • Mesh-free method — no background mesh; particles or radial basis functions carry solution data.

The interface coupling itself can further be classified by the type of mesh matching:

  1. Strong (conformal meshes) — fluid and solid meshes share nodes at the interface; velocity and traction continuity are enforced exactly.
  2. Weak / Nitsche method (non-conformal meshes) — interface conditions are enforced weakly via a penalty-like variational term, allowing independent fluid and solid mesh refinements.
😶
My own work uses the monolithic + body-fitted + fully coupled strategy with quasi-direct coupling.

Why ALE? Lagrangian vs. Eulerian

In classical fluid mechanics we work in an Eulerian frame: the computational domain is fixed in space and fluid flows through it. This is natural for fluids, which undergo large deformations and flow, but it makes tracking solid deformation awkward — we do not care which fluid particle is at a given point, only the field values there.

Solids, on the other hand, are naturally described in a Lagrangian frame: the mesh moves with the material. Individual material points are tracked, and displacements are physically meaningful. Using an Eulerian frame for solids would obscure the material identity needed for constitutive modelling.

When a deformable solid interacts with a fluid, we need a formulation that accommodates both: the Arbitrary Lagrangian-Eulerian (ALE) framework. In ALE, the computational mesh can move independently of the material — it moves with the solid boundary (Lagrangian) but deforms smoothly in the fluid interior (neither purely Eulerian nor purely Lagrangian). The governing equations gain an additional mesh velocity term that accounts for the moving reference frame.

ALE Solution Procedure

At each time step, an ALE-FSI solver performs two coupled sub-problems:

  1. Mesh motion problem — given the solid displacement field at the interface as a boundary condition, extend a smooth mesh deformation into the fluid domain. This is typically posed as a harmonic extension or a pseudo-elastic problem:
    ∇·(μm ∇w) = 0 in Ωf
    where w is the mesh displacement field and μm is a stiffening parameter (often inverse-proportional to element size to protect small elements near the interface).
  2. Moving-domain problem — solve the fluid and solid equations on the updated (deformed) mesh. The fluid momentum equation acquires an ALE convection term involving the mesh velocity ẇ:
    ρ(∂u/∂t|χ + (u − ẇ)·∇u) = −∇p + μ∇²u
    where the derivative ∂/∂t|χ is taken at a fixed ALE reference point χ, not a fixed material point or a fixed spatial point.
📌

Key insight: In the solid domain, the mesh displacement equals the physical material displacement — these coincide by definition. In the fluid domain, the mesh motion carries no physical meaning for the fluid material; it is a purely computational device that introduces a modified convection term into the Navier-Stokes equations.

Coupling Strategies

Once the three sub-problems (fluid, solid, mesh motion) are defined, they must be coupled. Common strategies, roughly ordered from loosest to tightest coupling:

  1. Block-iterative (Gauss-Seidel) method — fluid and solid are solved in sequence, exchanging interface data, and the loop is iterated to convergence within each time step. Robust but may converge slowly for strongly coupled problems (e.g. added-mass instability for light structures).
  2. Quasi-direct coupling — the Jacobian of the coupled system is approximated using the individual sub-problem Jacobians. Faster convergence than block-iterative with moderate implementation complexity. This is the approach used in my current solver.
  3. Direct (monolithic) strategy — fluid, solid, and mesh motion DOFs are assembled into a single system matrix and solved with a coupled preconditioner. Optimal convergence but demands a unified discretization and a sophisticated solver/preconditioner (e.g., nested block solvers with approximate Schur complements).

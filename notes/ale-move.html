ALE.move

Mesh Motion in FEniCS

In Fluid-Structure Interaction (FSI) or problems with moving boundaries, we often use the Arbitrary Lagrangian-Eulerian (ALE) formulation. In FEniCS, mesh deformation is handled by the ALE.move() function, which updates the vertex coordinates of a mesh based on a displacement field.

💡 Key Concept: The ALE method allows the mesh to move independently of the material (fluid), preventing excessive distortion that would occur in a pure Lagrangian approach.

Implementation Example

To move a mesh, you first solve for a displacement field (often using a harmonic or linear elastic extension) and then apply it:

import dolfin as df

# Define mesh
mesh = df.UnitSquareMesh(10, 10)

# Define a vector function space for displacement
V = df.VectorFunctionSpace(mesh, "CG", 1)
displacement = df.Function(V)

# ... Calculate displacement field (e.g., solve Laplace/Elasticity) ...

# Move the mesh
df.ALE.move(mesh, displacement)

# Optional: Update the boundary conditions and Jacobian if necessary
mesh.bounding_box_tree().build(mesh)

Harmonic Extension

A common way to determine the interior mesh displacement from a known boundary move is to solve the Laplace equation:

$$\Delta \mathbf{d} = 0 \text{ in } \Omega$$ $$\mathbf{d} = \mathbf{d}_{\text{boundary}} \text{ on } \Gamma$$

⚠️ Important: ALE.move modifies the mesh in-place. If you need the original mesh for comparison, ensure you create a Mesh(mesh) copy beforehand.

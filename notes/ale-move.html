ALE Mesh Motion

Category: FSI · Python · Mesh Motion · ALE

In Fluid-Structure Interaction (FSI) or problems with moving boundaries, we often use the Arbitrary Lagrangian-Eulerian (ALE) formulation. In FEniCS, mesh deformation is handled by the ALE.move() function, which updates vertex coordinates based on a vector displacement field.

Key Concept.

The ALE method allows the mesh to move independently of the material (fluid), preventing the catastrophic element distortion that would occur in a pure Lagrangian approach during large deformations.

Implementation Example

To move a mesh, you first calculate a displacement field (often using a harmonic extension) and then apply it directly to the mesh object:

import dolfin as df

# Define mesh and displacement space
mesh = df.UnitSquareMesh(20, 20)
V = df.VectorFunctionSpace(mesh, "CG", 1)
displacement = df.Function(V)

# ... Calculate displacement field (solve Laplace or Elasticity) ...

# Move the mesh in-place
df.ALE.move(mesh, displacement)

# Important: Update the bounding box tree for point queries
mesh.bounding_box_tree().build(mesh)

Harmonic Extension

A standard way to determine interior mesh displacement from known boundary motion is solving the Laplace equation:

\[ \Delta \mathbf{d} = 0 \text{ in } \Omega, \quad \mathbf{d} = \mathbf{d}_{\Gamma} \text{ on } \partial \Omega \]
In-Place Modification.

ALE.move modifies the mesh object itself. If you need to revert to the initial state, ensure you create a deep copy old_mesh = df.Mesh(mesh) before moving.

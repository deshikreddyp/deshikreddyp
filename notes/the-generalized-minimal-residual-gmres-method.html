The Generalized Minimal Residual (GMRES) method

GMRES stands for Generalized Minimal Residual. It is an iterative method for solving a linear system

Ax=bA x = bAx=b﻿

especially when:

  • AAA﻿ is large and sparse
  • AAA﻿ is not symmetric
  • direct methods like Gaussian elimination are too expensive

The main idea is simple:

  • start with an initial guess x0x_0x0​﻿
  • compute the residual r0=b−Ax0r_0 = b - A x_0r0​=b−Ax0​﻿
  • build better and better correction directions from

    r0,  Ar0,  A2r0,  …r_0,\; A r_0,\; A^2 r_0,\; \dotsr0​,Ar0​,A2r0​,…﻿

  • at step $k$, choose the approximate solution xkx_kxk​﻿ that makes the residual norm ∥b−Axk∥\|b-Ax_k\|∥b−Axk​∥﻿ as small as possible within that search space

That “search space” is called the Krylov subspace:

Kk(A,r0)=span{r0,Ar0,A2r0,…,Ak−1r0}K_k(A,r_0)=\text{span}\{r_0,Ar_0,A^2r_0,\dots,A^{k-1}r_0\}Kk​(A,r0​)=span{r0​,Ar0​,A2r0​,…,Ak−1r0​}﻿

So GMRES is called “minimal residual” because at each step it picks the best approximation in that subspace by minimizing the residual.

Why GMRES is useful:

  • it works well for nonsymmetric matrices
  • it only needs matrix-vector products, which is great for sparse systems
  • it often gives good approximations long before reaching the full dimension

The tradeoff:

  • each iteration needs orthogonalization against all previous basis vectors
  • memory and cost grow with iteration count

That is why people often use restarted GMRES, written as GMRES$(m)$, where after every mmm﻿ steps the process restarts.

Intuition

You can think of GMRES like this:

  • Jacobi or Gauss-Seidel use fixed update rules
  • gradient-type methods move in one “best local” direction
  • GMRES keeps collecting smarter directions generated by the matrix itself
  • then it asks: “Among all combinations of these directions, which one makes the residual smallest right now?”

That makes it very powerful for awkward nonsymmetric systems.

How GMRES is implemented in practice

GMRES usually uses the Arnoldi process to:

  • build an orthonormal basis of the Krylov subspace
  • convert the big problem into a much smaller least-squares problem

So instead of minimizing over a huge vector space directly, it solves a small optimization problem each iteration.

One very important practical idea is preconditioning:

instead of solving Ax=bAx=bAx=b﻿ directly, we solve a transformed system that has the same solution but converges faster. In real applications, preconditioning is often the difference between “GMRES is slow” and “GMRES works great.”

In one sentence

GMRES is an iterative Krylov-subspace method for large nonsymmetric linear systems that, at each step, chooses the approximation with the smallest residual over the space generated by r0,Ar0,A2r0,…r_0, Ar_0, A^2r_0, \dotsr0​,Ar0​,A2r0​,…﻿

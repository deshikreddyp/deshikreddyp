Dealing with Complex Geometries in CFD

Category: Meshing · CFD · Tools · GMSH

Last updated: ...

Processing Medical Geometries

Biomedical geometries are typically reconstructed from UV or PC MRI scans. PC MRI is particularly effective for capturing the fine three-dimensional details of an Area of Interest (AOI). Recent developments in 4D MRI have even surpassed standard computational meshing in resolution for certain applications.

Voxel data from MRI is connected using triangular facets to form a surface mesh. However, raw STL files resulting from this process lack connectivity and topological information, making them difficult to segment for imposing separate boundary conditions.

Conforming Meshes & Coupling

In computational fluid dynamics, solving coupled physical phenomena requires a Monolithic approach (strongly coupled). This approach necessitates a conformal mesh—one where the nodes on both sides of the interface (e.g., fluid and solid subdomains) match exactly.

GMSH Workflow.

Since most MRI sources are raw STL, the high-fidelity workflow involves:

  1. Convert STL to BRep/STEP (e.g., via stl2brep.py).
  2. Perform boolean operations (BooleanFragments) in GMSH.
  3. Specify physical groups for subdomains and boundaries.

GMSH to FEniCS Workflow

Modern FEniCS/FEniCSx workflows recommend using meshio for mesh conversion. Moving from GMSH $(.msh)$ to HDF5 $(.h5)$ is the most robust path for large-scale parallel simulations.

import dolfin as df
from mark_boundaries import *

def convert_mesh(prefix):
    # Standard legacy conversion logic
    os.system(f"dolfin-convert {prefix}.msh {prefix}.xml")
    mesh = df.Mesh(f"{prefix}.xml")
    
    # Load physical tags
    domains = df.MeshFunction("size_t", mesh, f"{prefix}_physical_region.xml")
    df.File("domains.pvd") << domains
Normal Inversion.

Converting between formats can randomly flip facet normals. Always use a utility like mark_boundaries.py to verify and fix boundary orientation before running the solver.

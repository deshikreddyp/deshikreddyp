Reference Values

Created@October 19, 2021 5:34 AM
Tagsansys

You might have often come up with a tab named “reference values” in software packages like Abaqus, Ansys. What are these values?

Usually problems are often non-dimentionalized for easier solving. For this, the system need some data like free-stream velocity, characteristic length, Pressure, Area, Density, Enthalpy, Temperature, Velocity, Specific Heats.

Ansys:

The reference values can be entered manually or computed based on physical quantity values at a specified boundary zone. Area, Density, Enthalpy, Length, Pressure, Temperature, Velocity, dynamic Viscosity, and Ratio Of Specific Heats are the reference quantities to be set.

Depth is an additional quantity that can be defined for 2D situations. This value will be used to report fluxes and forces, as well as variables computed using the Surface Integrals and Volume Integrals dialogue boxes (e.g. Area, Flow Rate, Mass Flow Rate, Volume, etc.). Before reporting any of the variables listed above, double-check that the value and units of Depth correspond to the depth dimension of your application.

In the Set Units dialogue box, the units for Depth are set separately from the units for Length.
Select the zone in the Compute From drop-down list if you wish to compute reference values from the conditions defined for a certain boundary zone. However, depending on the boundary condition, only a subset of the reference values may be set. The reference length and area, for example, will not be established by computing the reference values from a border condition; you must specify these manually.

To manually set the values, enter the value for each under the Reference Values heading.

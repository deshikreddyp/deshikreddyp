Reference Values

Category: CFD · Physics · Tools · Nondimensionalization

Last updated: ...

In industry-standard solvers like Abaqus and Ansys Fluent, "Reference Values" are characteristic quantities used for normalization, reporting nondimensional coefficients (like $C_d$ or $C_l$), and post-processing flux data.

Nondimensionalization.

Physical problems are frequently scaled to characteristic values to allow for easier comparison across different Reynolds numbers or scales. Typical reference data include free-stream velocity ($U_\infty$), characteristic length ($L$), area ($A$), and density ($\rho$).

Industry Standard: Ansys Fluent

In Fluent, the "Reference Values" task page allows you to specify quantities that populate equations for:

  • Force Coefficients: $C_d = \frac{F_d}{\frac{1}{2}\rho U^2 A}$.
  • Torque Coefficients: $C_m = \frac{M}{\frac{1}{2}\rho U^2 A L}$.
  • Flow Rates: Normalizing mass flow relative to a reference density.
2D Simulations.

For 2D simulations, the Depth (spanwise thickness) is a critical reference value. If the depth is left as the default $1.0$, force magnitudes ($N$) will be reported "per meter depth." Ensure this matches your physical problem to get correct force values.

Compute From...

While the "Compute From [Inlet]" button is convenient, it only populates values derived from that boundary condition (e.g., velocity, density). Geometric values like Reference Area or Length must almost always be entered manually based on your model's project area.

FEniCS: Interpolation across non-matching meshes

Interpolation Across Non-Matching Meshes

In multi-stage simulations or when switching from a coarse to a fine mesh, transferring solution fields is essential. This process, often called regridding, allows you to interpolate data from a source mesh to a target mesh that does not share the same topology.

💡 Key Components:

  • LagrangeInterpolator: The core FEniCS tool for high-performance interpolation between different meshes.
  • Extrapolation: Enabling set_allow_extrapolation(True) handles points near the boundaries that might fall slightly outside the source domain due to numerical tolerances.
  • HDF5 Snapshots: Efficient storage and retrieval of time-series data for batch processing.

Implementation Workflow

from dolfin import *
import os

# 1. Load Meshes
src_mesh = Mesh()
hdf_src = HDF5File(MPI.comm_world, "source.h5", "r")
hdf_src.read(src_mesh, "/mesh", False)

tgt_mesh = Mesh()
hdf_tgt = HDF5File(MPI.comm_world, "target.h5", "r")
hdf_tgt.read(tgt_mesh, "/mesh", False)

# 2. Define Function Spaces
V_src = VectorFunctionSpace(src_mesh, "CG", 1)
V_tgt = VectorFunctionSpace(tgt_mesh, "CG", 1)

# 3. Buffers and Interpolation
u_src = Function(V_src)
u_src.set_allow_extrapolation(True)
u_tgt = Function(V_tgt)

# Batch processing snapshots
snap_h5 = HDF5File(MPI.comm_world, "snapshots.h5", "r")
for i in range(64):  # Example: 64 snapshots
    snap_h5.read(u_src, f"/displacement/{i}")
    LagrangeInterpolator.interpolate(u_tgt, u_src)
    # Save u_tgt to new file...

⚠️ Precision: While interpolation preserves the mathematical form, mapping a high-order solution to a coarser mesh will result in loss of detail. Always verify energy or mass conservation after regridding.

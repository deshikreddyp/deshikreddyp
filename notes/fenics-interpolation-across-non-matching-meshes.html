Interpolation Across Non-Matching Meshes

Category: FEniCS

Last updated: ...

In multi-stage simulations or when switching from a coarse to a fine mesh, transferring solution fields is essential. This process, often called regridding, allows for the interpolation of data from a source mesh to a target mesh that does not share the same topology.

Key Components.
  • LagrangeInterpolator: The optimized FEniCS tool for interpolation between arbitrary meshes.
  • Extrapolation: Enabling set_allow_extrapolation(True) is often necessary to handle points near boundaries that might fall slightly outside the source domain due to numerical tolerances.
  • HDF5 I/O: High-performance storage is critical for handling large time-series data during batch regridding.

Implementation Workflow

from dolfin import *

# 1. Load Source and Target
src_mesh, tgt_mesh = Mesh(), Mesh()
# ... Read meshes from HDF5 ...

# 2. Define Function Spaces
V_src = VectorFunctionSpace(src_mesh, "CG", 1)
V_tgt = VectorFunctionSpace(tgt_mesh, "CG", 1)

# 3. Transfer Data
u_src = Function(V_src)
u_src.set_allow_extrapolation(True)
u_tgt = Function(V_tgt)

# Batch process snapshots
for i in range(n_snaps):
    # ... Load u_src data ...
    LagrangeInterpolator.interpolate(u_tgt, u_src)
    # ... Store u_tgt data ...
Resolution Loss.

Interpolation preserves the mathematical form but mapping a high-order or high-resolution solution to a coarser mesh results in a loss of spectral detail. Always verify conservation laws (e.g., mass balance) after the transfer.

HPC Considerations.

For large meshes, LagrangeInterpolator uses sophisticated point-search algorithms. In a parallel MPI environment, ensure the search trees are properly synchronized to avoid segmentation faults.
